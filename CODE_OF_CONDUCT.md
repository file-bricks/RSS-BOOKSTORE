# Verhaltenskodex / Code of Conduct

## Deutsch

### Unser Versprechen

Wir verpflichten uns, Beteiligung an diesem Projekt respektvoll, sachlich und frei von Belästigung zu halten. Das gilt unabhängig von Alter, Behinderung, Herkunft, Geschlechtsidentität, Erfahrungsgrad, Religion, sexueller Orientierung oder persönlichem Hintergrund.

### Erwartetes Verhalten

- Freundliche und inklusive Sprache verwenden
- Unterschiedliche Sichtweisen respektieren
- Kritik konkret, konstruktiv und projektbezogen formulieren
- Private Informationen anderer nicht veröffentlichen

### Nicht akzeptiertes Verhalten

- Beleidigungen, Drohungen, Trolling oder persönliche Angriffe
- Öffentliche oder private Belästigung
- Veröffentlichung privater Daten ohne ausdrückliche Zustimmung
- Wiederholtes Stören von Issues, Diskussionen oder Pull Requests

### Durchsetzung

Problematisches Verhalten kann über GitHub Issues oder, falls ein öffentlicher Hinweis ungeeignet wäre, über die GitHub-Profilkontakte des Maintainers gemeldet werden. Sicherheitsrelevante Meldungen gehören in das private Vulnerability-Reporting aus `SECURITY.md`.

## English

### Our Pledge

We are committed to keeping participation in this project respectful, constructive, and harassment-free for everyone, regardless of age, disability, ethnicity, gender identity, level of experience, religion, sexual orientation, or personal background.

### Expected Behavior

- Use welcoming and inclusive language
- Respect different viewpoints and experiences
- Keep feedback specific, constructive, and project-focused
- Do not publish private information about others

### Unacceptable Behavior

- Insults, threats, trolling, or personal attacks
- Public or private harassment
- Publishing private data without explicit permission
- Repeated disruption of issues, discussions, or pull requests

### Enforcement

Problematic behavior may be reported through GitHub Issues or, when a public report would be inappropriate, through the maintainer contact options on GitHub. Security-related reports should use the private vulnerability reporting process described in `SECURITY.md`.
